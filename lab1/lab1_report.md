Создание Telegram-бота без программирования

1. Описание 

В рамках лабораторной работы #1 был разработан Telegram-бот для управления задачами. Бот позволяет пользователю добавлять задачи в текстовом виде, назначать им приоритет, получать напоминания и управлять списком задач.
Функционал бота включает:
- добавление задач с указанием текста, приоритета и времени напоминания
- просмотр списка задач
- сортировку задач по приоритету
- удаление задач по номеру
- отправку напоминаний через заданное время

2. Какую проблему решает бот

Бот решает задачу организации личных задач в простом интерфейсе без необходимости использовать отдельные приложения (постоянно заходить в Заметки на Айфоне). Пользователь может быстро зафиксировать задачу прямо в Telegram, задать приоритет и получить напоминание.
Это снижает вероятность забыть о задачах и упрощает управление текущими делами (меньше стресса, круто)

3. Почему выбрана эта задача

Задача была выбрана, тк она соответствует актуальному пользовательскому запросу (моему) + хорошо демонстрирует возможности LLM при генерации рабочего кода

Функционал бота:
Бот является ассистентом для управления задачами (To-Do бот)
Основные возможности:
Пользователь может добавлять задачи через команду /add

Текст задачи
Приоритет (низкий / средний / высокий)

Задачи должны быть отсортированы по приоритету:
сначала высокий → затем средний → затем низкий
Команда /done принимает номер задачи и удаляет её
Напоминания:
Если указано время, бот отправляет сообщение через заданное количество минут
Использовать JobQueue
Дополнительно:
/start — описание
/help — инструкция
Требования:

7. Трудности и решения

Проблема 1: ошибка при работе напоминаний
При добавлении задачи с напоминанием возникала ошибка отсутствия JobQueue в библиотеке

Решение:
установка расширенной версии библиотеки:
pip install "python-telegram-bot[job-queue]"

Проблема 2: понимание процесса запуска

На начальном этапе было непонимание:
- где запускать код
- как работают файлы проекта
- как взаимодействует бот с Telegram

Решение:
настройка локального окружения (терминал на macOS) и последовательный запуск проекта
