## 2. Какой источник данных выбран

CSV файл


## 3. Почему выбран именно этот источник


* простой формат, не требующий настройки бд
* легко читается и редактируется
* подходит для реализации поиска и фильтрации


## 4. Структура данных

Используется файл `tasks_data.csv` со следующей структурой:

text, priority, created_at

Каждая строка соответствует одной задаче и содержит:

* текст задачи
* приоритет (низкий, средний, высокий)
* дату создания

## 6. Реализация

* при добавлении задачи данные записываются в файл
* при выполнении команды /list данные считываются и сортируются
* команда /search выполняет поиск по тексту задач
* команда /today фильтрует задачи по дате


Запись задачи в CSV:

```python
def save_task_to_csv(text, priority):
    with open(CSV_FILE, "a", encoding="utf-8", newline="\n") as f:
        writer = csv.writer(f)
        writer.writerow([text, priority, datetime.now().strftime("%Y-%m-%d")])
```

Чтение данных:

```python
def load_tasks_from_csv():
    with open(CSV_FILE, "r", encoding="utf-8") as f:
        reader = csv.DictReader(f)
        return list(reader)
```

Поиск:

```python
results = [t for t in tasks if keyword in t["text"].lower()]
```

Фильтрация по дате:

```python
today = datetime.now().strftime("%Y-%m-%d")
results = [t for t in tasks if t["created_at"] == today]
```

## 8. Трудности и решения

### Проблема 1: некорректная запись в CSV

При добавлении задач происходило объединение нескольких записей в одну строку, что приводило к ошибкам при чтении.

Решение:
использование параметра newline="\n" при открытии файла и корректная запись через csv.writer


### Проблема 2: некорректное отображение данных

Из-за поврежденной строки часть задач не отображалась в списке

Решение:
исправлена структура CSV и добавлена проверка корректности данных при чтении
